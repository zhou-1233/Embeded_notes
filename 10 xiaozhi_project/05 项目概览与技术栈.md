# 小智 AI (xiaozhi-esp32) 项目概览与技术栈

## 项目身份

- **项目名称**：小智 AI 聊天机器人 (xiaozhi-esp32)
- **GitHub 仓库**：`https://github.com/78/xiaozhi-esp32`
- **作者**：78（个人/社区开源项目，非乐鑫官方）
- **当前版本**：v2.0.4
- **定位**：基于 ESP32 的 AI 语音交互入口，利用 Qwen/DeepSeek 等大模型，通过 MCP 协议实现多端控制

## 核心功能

- Wi-Fi / ML307 Cat.1 4G 联网
- 离线语音唤醒（ESP-SR）
- 两种通信协议：WebSocket 或 MQTT+UDP
- OPUS 音频编解码
- 流式 ASR + LLM + TTS 语音交互
- 声纹识别（3D Speaker）
- OLED / LCD 显示屏，表情显示
- 电量显示与电源管理
- 多语言支持（40 种语言）
- 设备端 MCP 协议实现
- 支持 ESP32-C3、ESP32-S3、ESP32-P4 芯片平台
- 兼容 93 个开源硬件

## 技术栈全景图

### 🎯 底层：RTOS + 嵌入式基础

| 技术 | 用途 |
|---|---|
| **FreeRTOS** | 实时操作系统，多任务调度、事件组、消息队列、互斥锁 |
| **ESP-IDF** | 乐鑫官方开发框架 |
| **NVS** | WiFi 配置等持久化存储 |
| **I2C / I2S / SPI / PDM** | 嵌入式四大总线协议 |
| **CMake** | 构建系统 |

### 🎵 中间层：音频处理管线

| 技术 | 用途 |
|---|---|
| **OPUS 编解码** | 音频压缩，60ms 帧长 |
| **ESP-SR** | 乐鑫语音识别框架（唤醒词检测 + VAD + AEC） |
| **3D Speaker** | 声纹识别 |
| **ES8311 / ES7210 / ES8374 驱动** | 多种音频 Codec 芯片驱动 |
| **音频重采样** | 采样率转换 |

### 🌐 网络层

| 技术 | 用途 |
|---|---|
| **WebSocket** | 实时双向通信 |
| **MQTT + UDP** | 轻量级 IoT 消息协议 |
| **mbedTLS** | SSL/TLS 加密、Base64 编码 |
| **Wi-Fi (ESP-NETIF)** | 无线网络连接 |

### 🤖 应用层：AI + MCP

| 技术 | 用途 |
|---|---|
| **MCP (Model Context Protocol)** | Anthropic 提出的 AI-硬件交互协议 |
| **流式 ASR + LLM + TTS** | 云端语音识别→大模型→语音合成 |
| **cJSON** | 嵌入式 JSON 处理 |
| **LVGL** | 嵌入式 GUI 图形库，支持 GIF/JPG/表情/字体 |

### 🔌 外设驱动层

| 技术 | 用途 |
|---|---|
| **WS2812** | 可编程 RGB LED 灯珠驱动 |
| **XL9555** | I2C GPIO 扩展芯片 |
| **OLED (SSD1306) / LCD (ST7789/ILI9341 等)** | 多种屏幕驱动 |
| **OTA 固件升级** | 在线固件更新 |
| **AXP2101 / SY6970** | 电源管理芯片 |
