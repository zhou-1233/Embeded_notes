## 三、双非的现实影响

### 不利的一面
- 简历筛选阶段，985/211 有优势，尤其大厂
- 部分公司有硬学历门槛（如华为某些部门要求 211 起）
- 起薪可能比同水平 985/211 低 1-3K

### 有利的一面
- **嵌入式行业对学历的看重程度远低于纯软件**
- 嵌入式面试非常务实——"你做过什么"比"你哪个学校"重要得多
- 双非进嵌入式大厂拿好 offer 的案例非常多
- 能画 PCB + 写代码 + 调硬件，这个组合本身就稀缺

> **在嵌入式行业，能亮出一块自己画的 PCB + 跑着自己写的代码 + 能讲清楚每一行为什么这么写，面试官基本不会在乎你是什么学校。**

## 五、关键建议

### 1. 二次开发要有"亮点"

普通地加个传感器不算二次开发。**建议选一个有区分度的方向**，详见 [[05 学习路线与二次开发#三、二次开发方向]]。

### 2. Linux 不要拖到"以后学"

你目前技术栈最大缺口。从现在开始每天 1 小时：

| 阶段 | 内容 | 时间 |
|---|---|---|
| 第一阶段 | Linux 用户态编程（文件 IO、进程、线程、socket、epoll） | 1-2 月 |
| 第二阶段 | 内核基础（字符设备驱动、设备树、platform 驱动） | 1-2 月 |
| 第三阶段 | 系统构建（Buildroot/Yocto、uboot、内核裁剪） | 1 月 |

推荐教程：韦东山、正点原子。目标：能用 Linux 写多线程网络程序 + 看懂设备树 + 编译烧录 Linux 镜像。

### 3. 实习比项目更重要

一个大厂实习经历，比三个项目都值钱。找实习时：
- GitHub 项目 README 写好（英文），架构图 + 视频演示
- 大三上开学就投，大厂实习通道 9-10 月就开了
- 实习面试问的还是项目经历——吃透上面两个项目就够

### 4. 学会"技术叙事"

面试时不要只列技术名词，要学会讲故事：

> ❌ "我用 FreeRTOS 做了任务调度"
> 
> ✅ "这个项目有三条实时数据流——音频采集、OPUS 编码、网络发送，我用了三个 FreeRTOS 任务 + 四个队列来解耦它们。为什么分开？因为采集是 I2S 中断驱动的，编码是 CPU 密集的，放一起会互相阻塞..."
