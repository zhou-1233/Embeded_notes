**==C语言控制硬件的本质就是操作内存==**

--------
#### 编译过程
**1.预编译：** 展开头文件，替换宏定义，删除注释
**2.编译：** 把C语言翻译成汇编语言
**3.汇编：** 把汇编语言翻译成机器码
**4.链接：** 把多个机器码文件组合成可执行的程序

----------
**宏定义**
```C
#define  宏名   宏体
#define  XL9555_ADDR   0x20
```

--------------
**联合体：** 所有成员共享同一块内存空间，同一时间只能使用一个成员，内存大小等于最大成员的大小，优势是节省内存空间

------
**枚举：** 定义一组常量，常用于：如状态机
**typedef 数据类型 别名：** 给数据类型取别名
```c
typedef enum{
	idle,  //待机
	running,  //运行
	error,  //故障
}state;                     //typedef 数据类型 别名
```
-----------------------------------
**register：** 让编译器尽可能把变量放到寄存器里加速访问
**volatile：** 强制要求编译器每次都到内存中读取，禁止优化

**const:** 只读保护，定义的变量、参数、函数返回值不能被修改

----
##### static
**1. 限制作用域：** 没有加static的话，默认可被外部引用。所以在编程时要养成好习惯，默认静态，按需开发。把所有的全局变量和函数先用static修饰，只对外暴露必要的接口
**2.延长生命周期：** 普通的局部变量，每次调用函数时都会重新初始化，调用结束后销毁，而static修饰的局部变量程序结束后才销毁，用于：计数统计，缓存结果，保存状态，防止反复被初始化，解决局部变量每次调用都会重置的问题

**extern:** 声明外部变量或函数，实现跨文件访问，用static修饰的不能跨文件访问

---------------
#### 指针
**函数指针：** 

```c
//格式： 返回值类型 (*指针名) (形参列表)
void （*p1）() = method1;
 int (*p2) (int,int) = method2;

//给函数指针取别名       
//typedef  返回值类型  (函数指针别名)  (参数列表)
typedef void(*p_wifi_state_cb)(WIFI_STATE);
```

--------------
#### 位运算

**移位：** <<  >>
1. 快速乘除
2. 提取某些位
3. 循环移位

**按位与 & ：** 
1. 清零某些位
2. 提取某些位
3. 判断奇偶性

**按位或 | ：
1. 设置某些位
2. 合并两个数的某些位
3. 开启某些标志位

**按位异或 ^ :** 
1. 交换两个数
2. 判断两个数是否相等
3. 翻转某些位

**按位取反 ~ ：**
1. 取反操作
2. 计算负数
3. 掩码操作

------------
#### 内存操作

1. 函数访问：（）
2. 取值操作： []、 * 、 -> 、 .
3. 取地址： &
4. 内存打包： {}

-----------
#### 编程原则
1. **单一职能原则：** 一个函数/模块只负责一件事
2. **开闭原则：** 对拓展开放、对修改关闭
3. **里氏替换原则：** 子类可以替换父类，程序行为不变
4. **接口隔离原则：** 不要把不相关的功能塞到同一个接口中，每个模块只提供自己职责范围内的接口，用户用不到的就别暴露出来
5. **依赖倒置原则：** 别绑死在具体实现上，依赖抽象

**核心：** ==高复用、强边界==
